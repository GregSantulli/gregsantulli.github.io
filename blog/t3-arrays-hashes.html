T3: ARRAYS AND HASHES

In programming, we frequently deal with a collection of objects such as names from a database or entires of a blog. We can search, sort, filter, and futher manipulate these collections to meet out needs. This is all made possible by container objects in Ruby. The two most common container objects are Arrays and Hashes.

ARRAYS are ordered collections of objects. Each object is stored in a consistent, consecutive numerical index within the array. Consider the following array:

$ new_array = [1, 2, 3 , 4, 5]
Here we are creating an array consisting of the integers 1-5 and assigninging to to the variable new_array. One of the easiest ways to create an array is to just wrap a group of objects between brackets [ ] with each object separated by a comma. The order in which each object is listed determines each object's index. You can think of the an objects index as its position in the array. The first object in the array has an index of 0, the second has an index of 1, the third 2 and so fourth. This makes it east to locate an object based on its relative location within the array:

$ new_array = ["dev", "boot", "camp", "is", "my", "jam!"]
$ puts new_array[2]
>> "camp"
Each element in the array can be any object including numbers, strings, booleans, hashes, even other arrays. The contents of the array remain in the same exact order unless you explicitly add, remove, or move elements.

HASHES, on the other hand, store objects in pairs consiting of a key and a value. A common way to create a hash is with the literal constrictor, or curly braces: {}. Each key/value pair is assigned using the => operator, and each pair is separated by a comma. The following example shows a new hash with three key/value pairs:

$ new_hash = { 1 => "one", 2 => "two", "Name" => "Greg" }
Each key or value can be any object. Each key can be assigned multiple values, however the keys themselves must be unique to each hash. You can retrieve a value in a hash by looking up the corresponding key:

$ new_hash[1]
>> "one"
$ new_hash["Name"]
>> "Greg"
Array are like hashes where the keys are consecutive integers starting from 0. Hashes are like arrays where the indexes are allowed to be any object, not just integers.
