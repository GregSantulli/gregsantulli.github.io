T3: ARRAYS AND HASHES

In programming, we frequently deal with a collection of objects such as names from a database or entires of a blog. We can search, sort, filter, and futher manipulate these collections to meet out needs. This is all made possible by container objects in Ruby. The two most common container objects are Arrays and Hashes.

ARRAYS are ordered collections of objects. Ordered meaning that each object is stored in a consistent, consecutive numerical index within the array. Consider the following array:

$ new_array = [1, 2, 3 , 4, 5]
Here we are creating an array consisting of the integers 1-5 and assigninging to to the variable new_array. One of the easiest ways to create an array is to just wrap a group of objects between brackets [ ] with each object separated by a comma. The order in which each object is listed determines each object's index. You can think of the an objects index as its position in the array. The first object in the array has an index of 0, the second has an index of 1, the third 2 and so fourth. This makes it east to locate an object based on its relative location within the array:

$ new_array = ["dev", "boot", "camp", "is", "my", "jam!"]
$ puts new_array[2]
>> "camp"
