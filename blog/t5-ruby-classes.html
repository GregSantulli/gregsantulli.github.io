T5: RUBY CLASSES

In Ruby, pretty much everything we interact with is an object. All these objects are subject to methods that can be called upon them. In a large program, we are likely to have many different objects, so finding a way to organize these object and their respective methods might be a good idea.

Luckily, Ruby allows us to define Classes that not only allow us to organize objects and their methods, but also to assign specific attributes to each instance of an object belonging to a certain class. To explain this better, let's take a look at an example:

Let's say we're writing a program about farms. You might have objects such as farm and animal. Each of these objects might have different methods. Our animal object might take a method feed and our farm object might take somehting like cost_per_acre. These methods, however, cannot be applied to all of our objects. You cannot feed the farm itself and you certanly cannot calculate the acreage of an animal. To designate these methods to their respective objects, we can define classes as such:

class Farm
def cost_per_acre
#method definition goes here
end
 end

class Animal
def feed
#method definition goes here
end
end

The above example defines two classes: Farm and Animal. Inside each class, we defined a method that applies to only that class. If we tired to call feed on an instance of the Farm class, we would get an error (and vice versa). Speaking of instances, we can create a new instance of a class as follows:

sheep = Animal.new

The object sheep is now an instance of the Animal class and thus may have the feed method called upon it.

Another very useful feature of classes are their attributes. Instances of classes may be instantiated with attributes we define in an initialize definition:

class Farm
def initialize(owner, acres)
end
def cost_per_acre
#method definition
end
 end

class Animal
def initialize(name, color)
end
def feed
#method definition
end
end

Once defining neccessary attributes in the initialize definition, new class instances must be instantiated with all arguments in initialize, otherwise it will return an error:

sheep = Animal.new # Now Returns Argument Error
sheep = Animal.new("Lamb Chop", "white") # Ok

Each instance of a class will now contain these unique attributes passed to them upon initialization. This is useful because different anumals will usually have different attributes such as name, color, number of legs and so fourth.

As we know, according to the viariable scope, these attributes are only accessable inside the initialize definition. In order to make these attributes accessible outside of the initialize scope, we must assign them to instance variables:

class Animal
def initialize(name, color)
@name = name
@color = color
end
def feed
p "You just fed " + @name
end
end

Above, we assigned each argument of the initialize method to instance variables. Instance variables are defined with an "@" symbol in front and are usually only defined in the initialize method. We also wrote in a definiton for the feed method, which accesses the @name instance variable. Each instance variable is specific to each instance of a class (hence instance variable).

sheep = Animal.new("Lamb Chop", "white")
pig = Animal.new("Babe", "pink")

sheep.feed
=> "You just fed Lamb Chop"
pig.feed
=> "You just fed Babe"

As you might see, classes can be very useful when accessing information unique to each object. The prescence of classes and instance variables let certain methods know information specific to each instance of a class.
